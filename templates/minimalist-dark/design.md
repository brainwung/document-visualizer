# Minimalist Dark（极简深色风）

用于把文档、数据报告、会议纪要、论文讲解或运营复盘整理成深色模式下的 1080px HTML 长图。它是 `minimalist` 的深色版本，保留极简、讲义式、低装饰和高可读的结构，但所有颜色必须按深色模式映射处理。

## Source

本风格基于用户提供的《# 深色模式颜色对应 (1).md》整理。遇到未列出的浅色色值时，不要随意取近似深色；优先按语义选择本模板中已经定义的深色 token。

## When To Use

- 用户指定 `minimalist-dark`、深色模式、暗色长图、dark infographic。
- 内容需要在深色界面中阅读，尤其是数据报告、账号复盘、技术总结、会议纪要和论文讲解。
- 用户希望沿用 minimalist 的版式秩序，但页面整体必须是深色视觉。

## Visual System

- 浏览器/外层背景：`#121212`。
- 页面背景：`#121212`。
- 主模块背景：`#222222`。
- 次级模块背景：`#2C2C2C`。
- 更深卡片/输入类背景：`#353535`。
- 主文字：`#E0E0E0`。
- 正文辅助：`#A0A0A0`。
- 弱辅助/装饰英文：`#6C6C6C`。
- 主强调红：`#F04848`。
- 柔和强调底：`#4B2929`。
- 分割线/描边：优先 `#353535`；需要更强分割时用 `#5A5A5A`。
- 链接蓝：`#9ECDEE`。
- 黄色强调：`#FBA900`。
- 中文：AlibabaPuHuiTi。
- 数字：DIN 或同类数字字体。
- 辅助英文：AlibabaPuHuiTi Thin，字重 300。
- 普通正文行高：1.5。
- letter-spacing 保持 0。

## Structure

1. 顶部标题区：深色页面上使用清晰大标题，副标题和来源信息用 `#A0A0A0` 或 `#6C6C6C`，不要全部加粗。
2. 核心结论：可以使用 `#4B2929` 柔和强调底和左侧 `#F04848` 竖线；关键数字使用 `#F04848`。
3. 普通模块：使用 `#222222` 背景；如果模块内部还需要分组，用 `#2C2C2C`，不要继续套多层深灰。
4. 数据图表：图表层级必须少。外层已有 `#2C2C2C` 承载时，内部行不要再加独立背景。
5. 说明与口径：放在图表下方一行小字，使用 `#A0A0A0` 或 `#6C6C6C`，不要做成强卡片。
6. 底部总结：可使用 `#222222` 或更深的 `#121212`，但必须与页面边界有清晰间距或分割线。

## Module Rules

- 模块标题可采用“左侧中文标题 + 右侧 Thin 英文装饰”。右侧英文使用 `#6C6C6C`，字重 300，不承载必要信息。
- 主模块背景使用 `#222222`；同级模块背景必须统一。
- 模块内部卡片优先使用 `#2C2C2C`。只有需要强调的高光、风险或核心结论才使用 `#4B2929`。
- 同类信息卡必须共享字号、行高、padding、标题到正文间距和数字层级。可以通过底部红线或柔和红底表达强调，但不能破坏内部排版规则。
- 并列卡片必须等高，内部内容默认顶对齐；只有用户明确要求视觉居中时，才允许垂直居中。
- 标题到内容的距离应等于内容项之间的 gap。常用值为 18px。
- 不要使用强投影制造层级。深色模式中优先依靠背景层级、分割线和间距。

## Data Report Pattern

- 数据型页面必须先确认单位、口径、分母和时间范围。
- 不要在没有源文档明确证据时解释数据成因、平台机制、用户心理、内容策略效果或未来动作。
- 主指标使用 `#F04848`；辅助指标使用 `#E0E0E0` 或 `#A0A0A0`。
- Top 列表中，主指标放固定数字列，辅助指标放标题下方说明；不要为辅助指标再开一列造成重复。
- 绝对值柱状图不使用浅色进度底轨。只有比例、完成度或归一化比较才使用 `#4B2929` 底轨。
- 条形图底轨在深色模式下使用 `#4B2929`，数值和填充条使用 `#F04848`。
- 说明类文字、源表说明、口径说明和派生公式要弱化成常规小字，不要使用 `#4B2929` 大块承载。
- 日期本身重要时，不要弱化成难读小字；可以合并进标题，例如 `单日纪录·2025.11.07`。

## Accessibility

- 深色页面上正文不得低于 `#A0A0A0`，辅助英文或极弱元信息才使用 `#6C6C6C`。
- 红色文字 `#F04848` 只用于关键数字、重点路径、强调线或高光状态，不要整页大面积使用。
- `#4B2929` 只能做柔和强调底，不要在其上使用 `#6C6C6C` 小字；文字应至少使用 `#E0E0E0` 或 `#A0A0A0`。
- 分割线默认使用 `#353535`，不要用高亮红线做普通分割。

## QA

- 页面背景、模块背景、卡片背景是否分别是 `#121212`、`#222222`、`#2C2C2C/#353535`，层级不能混乱。
- 主文字、辅助文字、弱辅助文字是否分别落在 `#E0E0E0`、`#A0A0A0`、`#6C6C6C`。
- 企业红是否使用 `#F04848`，柔和红底是否使用 `#4B2929`。
- 是否存在浅色模式残留：`#F9F9F9`、`#FFFFFF`、`#F3F3F3`、`#FFEDEB`、`#E62828`。
- 图表、表格和卡片是否层级过多；有外层深灰承载时，内部不要继续套白底或浅底。
- 是否遵守 `RULES.md` 的数据真实性、禁止项和用户确认规则。
