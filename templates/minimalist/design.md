# Minimalist（极简风）

用于发言总结、培训讲义、会议长文提炼、演讲笔记、段落型报告等场景。目标是把长文本整理成清晰、完整、可读性高的 1080px HTML 长图。风格关键词：极简、段落讲义、清晰层级、低装饰、高可读。

## Preview

参考预览图：`assets/styles/minimalist.png`。预览图只用于理解大致风格，不是固定模板。

## When To Use

- 源材料是口语化发言、培训转写、会议记录、长段落笔记，而不是强数据表或复杂架构图。
- 用户希望“清晰、完整、可读性高”，内容需要按主题段落连续阅读。
- 图形不应过度卡片化；更适合极简讲义式段落：一个总纲 + 多个大段落模块。

## Visual System

- 画布：1080px 宽，高度随内容展开。
- 背景：`#F9F9F9`。
- 模块：`#FFFFFF`，不使用描边，用背景、投影和间距区分层级。
- 强调：`#e62828`；柔和强调底：`#ffedeb`。
- 字色：`#333333` / `#666666` / `#999999`。
- 中文：AlibabaPuHuiTi；数字编号：DIN；辅助英文：AlibabaPuHuiTi Thin。
- 正文行高：1.5。
- 间距使用 6px 倍数，常用 24px、30px、36px、60px、66px。

## Structure

1. 顶部标题区：标题、简短副标题、来源或日期。
2. 核心结论：柔和强调底色，保留左侧红色竖线，和后续普通段落模块形成区分。
3. 主题段落模块：每个模块由标题色条和白底正文组成。
4. 过程/关系模块：可少量使用流程、双栏、编号条目，但不要破坏段落阅读节奏。
5. 来源与待确认：把源文档缺失的信息单独说明，不编造日期、参会人、负责人或截止时间。
6. 底部总结：用深色模块收束全篇。

## Module Rules

- 标题色条：左侧放中文主标题，右侧放大数字编号和辅助英文。
- 右侧数字：DIN 粗体，保持强识别。
- 右侧英文：AlibabaPuHuiTi Thin，字重 300，不能抢中文标题层级。
- 正文主体：白底无边框，使用投影区分模块。建议投影 `0 18px 42px rgba(51,51,51,.07)`。
- 段落条目：小红竖条 + 粗标题 + 正文说明。竖条用于扫描，不要把每条都做成独立卡片。
- 核心结论：保留左侧红色竖线，表示它是总纲。
- 新增模块必须延续已建立的模块标题样式、分割线、间距和背景规则；例如同级模块都应使用同一套标题字号、右侧辅助说明和标题下分割线，不要单独改成另一种标题层级或过重背景。
- 若采用黑底模块标题条，标题使用白色，右侧辅助文字使用白色低透明度；黑底标题条只用于同级模块标题，不要把整个内容模块都改成黑底，除非内容资产在黑底上更清晰。
- 同级模块标题条到内容的距离必须等于内容项之间的 gap。常用值为 18px：标题条到首个条目 18px，条目之间 18px，logo 网格横纵间距 18px。
- 大模块之间用 60px 或 66px 拉开；标题条内部、卡片内部、logo 网格内部使用更小间距，避免层级混淆。

## Event Poster Pattern

适用于活动介绍、嘉宾长图、参会机构展示等场景。

- 活动标题、时间、地点、简介、嘉宾列表和参会机构应尽量放在一个主白色模块中，表示它们属于同一张活动信息海报；不要把时间地点和嘉宾拆成两个互不相关的大模块。
- 人物条目可以使用“左侧方图 + 右侧浅灰信息块”的结构。桌面端左图和右侧信息块应同高、同顶边、同底边；例如头像 `204px × 204px`，右侧信息块也为 204px 高。
- 人物信息块 padding 必须上下对称、左右对称，并使用 6px 倍数；例如 `18px 36px`。
- 人物姓名和序号可以同一行：姓名左对齐，序号右对齐；两者字号一致时序号仍应只作为检索辅助，使用品牌红 `#e62828`，不要再增加额外装饰线。
- title 与机构行属于次级信息，字号应低于姓名；机构行上方分割线使用浅灰，不使用红色，避免误读成重点标注。
- logo 模块必须先确认资源在当前背景上的可读性。若来源 logo 是白色透明版本，不要放浅底；若 logo 是黑色或深色，不要放黑底。必要时换用来源页同组里更适合当前底色的 logo。
- logo 网格适合使用浅灰底单元格承载，每个单元格高度固定，logo 只做等比 contain，不拉伸、不裁切。桌面可用 4 列，手机可降为 2 列。
- 对来源网站资源，应优先下载到本地 assets 再引用，避免预览时依赖远程资源；同时保留真实 alt 文本。

## Content Rules

- 先提炼观点，再放例子。不要把原文流水账整段塞进长图。
- 每个大模块只承载一个主题，例如“培训起点”“为什么会有后端”“思维转变”“案例实践”。
- 每个段落条目包含一个强标题和一段解释，解释尽量保留源文档里的关键例子和术语。
- 没有来源支持的日期、数字、负责人、截止时间、参会人，不要补造；标为“源文档未提供”。

## Double Column Alignment

- 双栏模块如果左右各有相同数量条目，必须按行对齐。
- 不要让左右列表各自自然流式排列，容易出现 A/B/C 与 2xx/4xx/5xx 错行。
- HTML 中优先使用显式行容器，例如每一行一个 `.aligned-row`，内部左右两列分别放对应条目。
- 同级条目使用同一套编号形态：A/B/C 或 01/02/03。
- 左右列宽使用 `grid-template-columns: 1fr 1fr`，列间距使用 24px 或 30px。

## QA

- 1080px 页面宽度稳定。
- 无横向溢出。
- 大模块宽度一致。
- 辅助英文使用 Thin。
- 双栏同级条目逐行对齐。
- 核心结论和普通模块有清晰区分。
- 无调试标记、占位文本、红框标注或 TODO。
