# Document Visualizer Rules

这些规则适用于所有 `document-visualizer` 输出。模板只负责视觉风格；事实完整性、字体、颜色、结构和 QA 必须遵守本文件。

## 信息完整性

- 不编造事实、指标、日期、负责人、截止时间、引用或来源。
- 区分源文档事实、整理分析、假设和待确认问题。
- 保留源文档里的关键数字、标签、日期、实体、文件路径和原始术语。
- 数据型文档必须先确认单位、口径、分母和时间范围；缺少这些信息时不要制作可能误导的图表。
- 会议纪要必须分离决策、待办、阻塞点和待确认项。负责人或日期缺失时标为“未提供”，不要补造。
- 口语化发言可以整理成书面语，但不能改变原意；关键态度和关键例子应保留。

## 视觉模式选择

| 源材料结构 | 推荐模式 | 避免 |
| --- | --- | --- |
| 时间顺序、项目历史、事故经过 | 时间线、里程碑、泳道时间线 | 泛化卡片网格 |
| 流程、审批、用户旅程、算法步骤 | 流程图、旅程图、状态机 | 密集段落摘要 |
| 系统模块、集成、技术方案 | 架构图、分层图、数据流 | 装饰性网络图 |
| 竞品、方案、取舍 | 对比矩阵、象限、评分表 | 无解释雷达图 |
| 策略、层级、能力模型 | 金字塔、分层模型、树图 | 随机图标 |
| 论文、研究方法 | 研究流程、claim-evidence、实验矩阵 | 只有摘要文本的海报 |
| 会议纪要 | 决策板、行动清单、风险表、时间线 | 逐字稿长文 |
| 数据报告 | KPI 条、柱/线/散点、标注表格 | 多分类饼图 |
| 发言总结、培训讲义 | 段落长图、主题模块、要点讲义 | 过度碎片化卡片 |

## 通用版式

- 长图推荐 1080px 宽，高度根据内容展开。
- 文档配图推荐 1600x900 或 1200x675。
- PPT 16:9 推荐 1600x900 或 1920x1080。
- 外边距：1080px 长图用 48-72px；1600px 画布用 64-96px。
- 模块间距：40-72px，或按模板约定的 6px 倍数执行。
- 模块 padding：24-36px。上下、左右留白必须看起来均衡。
- 使用严格对齐逻辑。即使版式有表现力，也要能看出网格。
- 新增模块或新增示例时，必须先复用当前页面已经建立的标题、分割线、间距、颜色和层级规则；除非用户明确要求新风格，不要为新内容临时引入一套新的模块标题或强背景。
- 同级模块标题必须使用同一套结构和视觉层级：标题字号、右侧辅助文案、背景/分割线、内边距和标题到内容的间距都要一致。若尝试黑底标题条，必须应用到所有同级模块，而不是只给新增模块使用。
- 标题到内容的间距应和内容项之间的间距保持一致。例如列表 gap 为 18px，则标题条到第一项也应为 18px；logo 网格横纵 gap 为 18px，则标题到 logo 网格也应为 18px。
- 大模块之间的分隔距离要明显大于模块内部项目间距，推荐 60px 或 66px；不要让段落、标题条、列表项之间的距离混成一个层级。
- 不要保留用户截图里的红框、箭头涂鸦、调试边框或临时标注。

## 字体与颜色

- 默认背景：`#F9F9F9`。
- 默认模块：`#FFFFFF`。
- 主文字：`#333333`。
- 正文辅助：`#666666`。
- 弱辅助/英文辅助：`#999999`。
- 主强调：`#e62828`。
- 柔和强调底：`#ffedeb`。
- 中文：AlibabaPuHuiTi。
- 数字：DIN 或同类数字字体。
- 辅助英文：AlibabaPuHuiTi Thin，字重 300，不要抢中文主标题层级。
- 普通正文行高：1.5。
- letter-spacing 保持 0。
- 不要用单一色相堆满页面；强调色只用于关键结构、编号、重点路径或总纲。
- 深色背景只用于需要强调的标题条、底部总结或明确的深色模块。若内容包含 logo、图片或品牌资产，必须先确认可读性，不要让黑色 logo 落在黑底、白色 logo 落在白底。

## SVG/HTML 输出

- SVG 要有 `width`、`height` 和 `viewBox`。
- SVG 核心文字尽量保留 `<text>`，不要转路径。
- SVG 使用简单 shape、line、rect、text，避免不必要的 raster。
- HTML 适合长文本、复杂换行、需要用户迭代样式的长图。
- 用户要求先看 HTML 时，先交付 HTML；不要提前导出 PNG。
- 若生成内部 QA 截图而用户不需要图片，检查后删除临时截图。

## QA 清单

- 页面/画布是否在目标尺寸内稳定显示。
- 是否有横向溢出、文字遮挡、截断或碰撞。
- 大模块宽度、标题条、正文块是否对齐。
- 同级模块标题样式是否一致；新增模块是否复用了已有规范。
- 标题到内容的间距是否等于内容项之间的间距。
- 大模块之间的间距是否明显大于模块内部间距。
- 文字层级是否清楚，英文辅助是否轻于中文标题。
- 重要数字、日期、负责人、路径和标签是否来自源文档。
- 假设和未知项是否和事实分开。
- 流程箭头是否方向明确、与模块保持可见间距。
- 双栏同级条目是否逐行对齐。
- 是否存在 TODO、placeholder、lorem、debug、红框标注或命令输出。
